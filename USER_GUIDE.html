1. Overview1. Обзор

DreamCatcher is a server-rendered jewelry shop built on FastAPI + SQLite + Jinja2 (no Node/Docker). Visitors browse the catalog, add items to a cart, check out, and pay online via Viva.com. The administrator manages products, categories, orders and site settings. The site runs in two languages: Greek (default) and English.

DreamCatcher — это магазин украшений с серверным рендерингом на FastAPI + SQLite + Jinja2 (без Node/Docker). Посетитель просматривает каталог, кладёт товары в корзину, оформляет заказ и оплачивает онлайн через Viva.com. Администратор управляет товарами, категориями, заказами и настройками. Сайт работает на двух языках: греческий (по умолчанию) и английский.

2. The customer journey2. Путь покупателя

  1. Browse the catalog or a product page. Priced, in-stock items show an Add to cart button; items marked “price on request” show a contact button instead.
  2. Просмотр каталога или карточки товара. У товаров с ценой и в наличии есть кнопка «В корзину»; у товаров «цена по запросу» — кнопка связи.
  3. Cart — change quantity, remove items. Prices are shown gross (VAT included); the VAT share is displayed.
  4. Корзина — изменить количество, удалить позиции. Цены брутто (с НДС); доля НДС показывается отдельно.
  5. Checkout — enter contact + shipping details, pick a shipping method.
  6. Оформление — контакты + адрес доставки, выбор способа доставки.
  7. Pay — redirected to the payment page. On success the order is confirmed.
  8. Оплата — переход на страницу оплаты. При успехе заказ подтверждается.

3. Admin: products, orders, settings3. Админка: товары, заказы, настройки

Open /admin/login and sign in (single admin account). The dashboard shows product and order counts.

Откройте /admin/login и войдите (единый аккаунт администратора). На дашборде — счётчики товаров и заказов.

Orders

Заказы

The Orders section lists every order with a status filter (pending → paid → shipped → cancelled). Open an order to see line items, totals, customer and shipping details, and payment references. Change status there (e.g. mark a paid order as shipped, or cancel it). Payment to paid happens automatically via the payment webhook — not by hand.

Раздел «Заказы» показывает все заказы с фильтром по статусу (pending → paid → shipped → cancelled). Откройте заказ — увидите позиции, суммы, контакты и адрес, данные платежа. Там же меняется статус (например, оплаченный → отправлен, или отмена). Перевод в paid происходит автоматически по webhook оплаты — не вручную.

Products & settings

Товары и настройки

Manage products (price, stock, translations, images) as before. In Settings set the VAT rate (Greece 24%), shipping methods (one per line: slug|Greek label|English label|cost) and the order notification email.

Товары (цена, остаток, переводы, изображения) управляются как раньше. В настройках задаются ставка НДС (Греция 24%), способы доставки (по строке: slug|метка el|метка en|стоимость) и email уведомлений о заказах.

4. Payments (Viva)4. Оплата (Viva)

Payments go through a provider abstraction, so another gateway (e.g. Stripe) can be added later without rewriting orders. Two modes via VIVA_MODE:

Оплата идёт через абстракцию провайдера, поэтому позже можно подключить другой шлюз (напр. Stripe) без переписывания заказов. Два режима через VIVA_MODE:

VIVA_MODEBehaviourПоведение
demo Internal stub checkout — no keys needed. The demo pay page calls the same webhook path the real gateway would, so the flow is genuine. Внутренняя заглушка — ключи не нужны. Страница demo-оплаты дёргает тот же webhook, что и реальный шлюз, — поток настоящий.
live Real Viva Smart Checkout. Requires merchant keys in .env (skeleton present; confirm API fields against Viva docs before enabling). Реальный Viva Smart Checkout. Нужны ключи мерчанта в .env (каркас готов; перед включением сверить поля API с документацией Viva).
The webhook is the only proof of payment. Returning to the “success” page does not mark an order paid. The webhook is idempotent: a redelivered event will not pay an order or reduce stock twice. Единственное подтверждение оплаты — webhook. Возврат на страницу «успех» не переводит заказ в оплаченный. Webhook идемпотентен: повторная доставка события не оплатит заказ и не спишет остаток дважды.

5. Running locally5. Локальный запуск

cd D:\Dreamcatcher_site_with_billing
python -m venv .venv; .\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env      # set SECRET_KEY, ADMIN_PASSWORD_HASH; VIVA_MODE=demo
python -m app.seed          # demo data
python -m pytest -q         # 14 tests should pass
python -m uvicorn app.main:app --reload   # http://127.0.0.1:8000 -> /el/
For local HTTP, keep DEBUG=true (or SESSION_HTTPS_ONLY=false) so the session cookie — which holds the cart — works without HTTPS. In production leave it on. Для локального HTTP держите DEBUG=true (или SESSION_HTTPS_ONLY=false), чтобы cookie сессии (в нём корзина) работала без HTTPS. В проде — включить.

6. Publishing a demo6. Публикация демо

Quick & free (Cloudflare Tunnel): run the dev server, then expose it with a temporary public HTTPS link — no account needed (works while your PC is on):

Быстро и бесплатно (Cloudflare Tunnel): запустите сервер и откройте его временной публичной HTTPS-ссылкой — без аккаунта (работает, пока включён ваш ПК):

# terminal 1 — server (session cookie over the tunnel's HTTPS)
SESSION_HTTPS_ONLY=false python -m uvicorn app.main:app --port 8000
# terminal 2 — tunnel
cloudflared.exe tunnel --url http://127.0.0.1:8000
# -> prints https://<random>.trycloudflare.com

For a persistent free link that survives your PC being off, deploy to Render (free web service, needs a GitHub repo) or Hugging Face Spaces (Docker space). On free tiers SQLite is ephemeral — fine for a showcase (re-seed on boot).

Для постоянной бесплатной ссылки, не зависящей от вашего ПК, разверните на Render (free web service, нужен GitHub-репозиторий) или Hugging Face Spaces (Docker space). На бесплатных тарифах SQLite эфемерный — для демо нормально (пересев при старте).

7. Important notes7. Важные детали

  • Prices are gross (VAT-inclusive); VAT is back-computed: vat = total − total/(1 + rate/100).
  • Цены брутто (с НДС); НДС считается обратным ходом: НДС = итог − итог/(1 + ставка/100).
  • Cart lives in the server session (only ids + quantities); prices are always read from the DB and snapshotted onto the order at checkout.
  • Корзина хранится в серверной сессии (только id + количества); цены берутся из БД и фиксируются в заказе при оформлении.
  • Backups: python -m scripts.backup copies data.db and static/uploads/ into backups/<timestamp>/.
  • Бэкап: python -m scripts.backup копирует data.db и static/uploads/ в backups/<timestamp>/.
  • Screenshots are regenerated with python scripts/shoot.py (needs Playwright + a running dev server).
  • Скриншоты пересоздаются командой python scripts/shoot.py (нужен Playwright и запущенный сервер).
  • The billing spec is SPEC-BILLING.md; the showcase spec remains SPEC.md.
  • Спека по биллингу — SPEC-BILLING.md; спека по витрине — SPEC.md.
