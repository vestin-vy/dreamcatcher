# DreamCatcher — Техническое задание (SPEC)

> Сайт-витрина ювелирного магазина **DreamCatcher**. Локальный запуск, рабочая
> CRUD-админка, мультиязычность (греческий по умолчанию).
> Версия SPEC: 1.0 · Дата: 2026-06-20 · Стек: Python (FastAPI + Jinja2 + SQLite).
>
> Этот файл — источник истины для реализации. Реализовывать рекомендуется в
> **отдельной чистой сессии** в plan mode: «Реализуй проект по `D:\DreamCatcher\SPEC.md`».

---

## 1. Цель и формат

DreamCatcher — это **витрина-визитка** (не интернет-магазин): красивый каталог
украшений с фотографиями и описаниями, плюс закрытая админка для наполнения
контентом. Покупки онлайн нет — целевое действие посетителя — **связаться**
(форма/телефон/мессенджер).

**Ключевые решения (зафиксированы на интервью):**

| Параметр | Решение |
|---|---|
| Модель | Витрина + контакт (без корзины/оплаты) |
| Стек | FastAPI + Jinja2 + SQLite (Python 3.12, без node/npm/docker) |
| Админка | Полная CRUD: товары, фото, переводы, видимость, заявки нет |
| Языки | **el (греческий) — по умолчанию**, en, ru, fr |
| Цены | Показывать (валюта по умолчанию EUR) |
| Дизайн | Светлый luxury (Cormorant + Montserrat, золотой акцент) |
| Страницы | Главная, Каталог, Карточка товара, О нас/Бренд, Контакты+карта, Коллекции |

## 2. Технологический стек и ограничения среды

> 🔴 **Жёсткое ограничение машины:** есть только **Python 3.12**. Нет node/npm/docker.
> Команда интерпретатора — **`python`** (не `python3` в скриптах сборки на чистой
> машине; хотя `python3` теперь тоже работает через шим).

| Слой | Выбор | Обоснование |
|---|---|---|
| Веб-фреймворк | **FastAPI** + Uvicorn | Современный, быстрый, типобезопасный, async |
| Шаблоны | **Jinja2** (server-side rendering) | Без сборки фронта, без node |
| БД | **SQLite** (файл `data.db`) | Ноль администрирования, портативно |
| ORM | **SQLModel** (SQLAlchemy + Pydantic) | Чистые модели, миграции через создание схемы |
| Стили | **Рукописный plain CSS** (без Tailwind, без сборки) | Решено на пре-флайте: ноль сборки, нет node. Авторим из `static/css/tokens.css` (переменные из MASTER.md). НЕ подключать Tailwind/PostCSS/Play CDN. |
| Картинки | **Pillow** | Ресайз, генерация превью, конвертация в WebP |
| Формы/валидация | Pydantic + Jinja2 | — |
| Авторизация | Сессионная cookie (itsdangerous) + bcrypt-хэш пароля (passlib, **bcrypt<4.1**) | Одна учётка администратора. bcrypt 4.1+ ломает version-probe passlib на 3.12 |
| i18n | **Babel** + .po/.mo ИЛИ простой JSON-словарь | UI-строки; контент — из БД |
| Карта | OpenStreetMap (Leaflet, CDN) или Google Maps embed | Без API-ключа предпочтительно Leaflet |

**Зависимости (pip, `requirements.txt`):**
```
fastapi
uvicorn[standard]
jinja2
sqlmodel
python-multipart        # загрузка файлов
itsdangerous            # подпись сессий
passlib[bcrypt]         # хэш пароля
bcrypt<4.1              # совместимость с passlib на 3.12
pillow                  # обработка изображений
babel                   # i18n (или обойтись JSON-словарём)
python-dotenv           # конфиг из .env
pytest                  # smoke-тесты (tests/test_smoke.py)
httpx                   # клиент для FastAPI TestClient
```

> ⚙️ Версии в `requirements.txt` зафиксированы (pinned) для воспроизводимости на
> ограниченной машине. Стили — **plain CSS** (Tailwind не используется, см. таблицу выше).

## 4. Модель данных (SQLite / SQLModel)

```
Category
  id            PK
  slug          unique           # rings, necklaces, earrings, bracelets...
  sort_order    int
  is_active     bool
  translations  -> CategoryTranslation[]   (name по языкам)

CategoryTranslation
  id, category_id FK, lang (el|en|ru|fr), name

Product
  id            PK
  slug          unique           # генерируется из en-названия или транслита
  sku           str nullable
  category_id   FK
  price         decimal nullable
  currency      str default 'EUR'
  price_on_request bool default false   # если true — показывать «по запросу»
  is_active     bool default true       # видимость на сайте
  is_featured   bool default false      # для блока «Коллекции/Хиты» на главной
  sort_order    int
  created_at, updated_at
  images        -> ProductImage[]
  translations  -> ProductTranslation[]

ProductTranslation
  id, product_id FK, lang (el|en|ru|fr)
  title         str
  description   text
  material      str nullable     # «золото 585», «серебро 925» и т.п.
  -- (любые маркетинговые поля, требующие перевода)

ProductImage
  id, product_id FK
  filename      str              # путь в static/uploads
  thumb         str              # путь превью
  alt           str nullable
  sort_order    int              # первая = главная
  width, height int

Setting   (key-value, одна строка конфигурации сайта)
  key           PK               # site_title, phone, email, address, whatsapp, instagram,
                                 # map_lat, map_lng, default_lang, show_prices ...
  value         str
```

**Правила:**
- Каждый `Product` обязан иметь перевод на язык по умолчанию (el). Остальные языки —
  опциональны; при отсутствии перевода **фолбэк на el** (затем en).
- Удаление товара — мягкое (`is_active=false`) ИЛИ жёсткое с удалением файлов фото
  (уточнить в реализации; по умолчанию — жёсткое с очисткой `static/uploads`).

## 5. Мультиязычность (i18n)

- **Языки:** `el` (по умолчанию), `en`, `ru`, `fr`.
- **Стратегия URL:** префикс языка — `/el/...`, `/en/...`, `/ru/...`, `/fr/...`.
  Корень `/` редиректит на `/el/` (или на язык из cookie, если выбран ранее).
- **Две категории строк:**
  1. **UI-строки** (меню, кнопки, подписи форм, футер) — словари `i18n.py`
     (или Babel .po). Заполнить для всех 4 языков сразу.
  2. **Контент** (названия/описания товаров, категории) — из БД, поле на каждый язык,
     с фолбэком на `el`.
- **Переключатель языка** в навбаре — сохраняет выбор в cookie, переключает,
  оставаясь на той же странице.
- **SEO:** `<html lang>`, `hreflang`-альтернейты, переведённые `<title>`/`meta`.
- **Форматирование:** цены/даты через Babel-локаль (EUR по умолчанию).

## 6. Страницы и сценарии (публичная часть)

Порядок секций главной — по паттерну ui-ux-pro-max (Feature-Rich Showcase),
адаптирован под витрину:

### 6.1 Главная (`/{lang}/`)
1. **Hero** — крупная фотография/слоган бренда, логотип DreamCatcher, мягкая
   анимация, CTA «Смотреть каталог».
2. **Избранные коллекции / Хиты** — 4–6 карточек (`is_featured`).
3. **Категории** — плитки (кольца, серьги, колье...).
4. **О бренде (тизер)** — короткий блок со ссылкой на «О нас».
5. **CTA-блок** — «Связаться» (телефон/WhatsApp/форма).

### 6.2 Каталог (`/{lang}/catalog`)
- Сетка карточек товаров; фильтр по категории; сортировка (новизна/цена).
- Карточка: главное фото, название, цена (или «по запросу»), hover-эффект.
- Адаптив: 1/2/3/4 колонки на 375/768/1024/1440.

### 6.3 Карточка товара (`/{lang}/product/{slug}`)
- Галерея фото (главное + миниатюры, lightbox), название, материал, описание, цена.
- Кнопка **«Запросить / Связаться»** → форма/мессенджер с предзаполненным товаром.
- Блок «Похожие из категории».

### 6.4 О нас / Бренд (`/{lang}/about`)
- История DreamCatcher, философия, мастерство; редактируемый текст (Setting/страница).

### 6.5 Контакты (`/{lang}/contact`)
- Адрес, телефон, email, соцсети (Instagram/WhatsApp), **карта** (Leaflet/OSM по lat/lng
  из Settings), **форма обратной связи** (имя, контакт, сообщение).
  Форма — отправка на email (SMTP) ИЛИ просто `mailto:`/WhatsApp-ссылка в MVP.

### 6.6 Коллекции (`/{lang}/collections`)
- Тематические подборки/«Новинки»/«Хиты» (на основе `is_featured`/категорий).

## 7. Дизайн-система (из ui-ux-pro-max)

> Источник: `design-system/dreamcatcher/MASTER.md`. Выбранное направление —
> **Светлый luxury / editorial** (минимализм, много воздуха, фото-центрично).
> ⚠️ Стиль «Liquid Glass» из автогенерации НЕ берём как основной (движок сам
> помечает его производительность/контраст как риск) — используем лёгкие
> glass-акценты точечно (навбар, модалки), не более.

### Палитра (CSS-переменные → `static/css/tokens.css`)
```css
:root{
  --color-primary:#1C1917;      /* премиум-чёрный (текст, кнопки-контур) */
  --color-on-primary:#FFFFFF;
  --color-secondary:#44403C;
  --color-accent:#A16207;       /* золото (CTA, акценты; WCAG-safe) */
  --color-background:#FAFAF9;   /* тёплый белый */
  --color-foreground:#0C0A09;
  --color-muted:#E8ECF0;
  --color-border:#D6D3D1;
  --color-destructive:#DC2626;
  --color-ring:#1C1917;
  /* spacing: 4/8/16/24/32/48/64; shadows sm..xl — см. MASTER.md */
}
```

### Типографика
- **Заголовки:** Cormorant (400–700) — серифный, ювелирный, элегантный.
- **Текст:** Montserrat (300–700).
- Подключение: Google Fonts (`@import` из MASTER.md) либо локальные .woff2 для офлайна.
- Базовый размер 16px, line-height 1.5, крупные «воздушные» заголовки.

### Компоненты
- Кнопки (primary золото / secondary контур), карточки с мягкой тенью и hover-lift,
  инпуты с золотым focus-ring, модалки с лёгким backdrop-blur — спеки в MASTER.md §Component Specs.

### Обязательный чек-лист перед сдачей (из MASTER.md)
- [ ] Иконки — SVG (Heroicons/Lucide), **не эмодзи**.
- [ ] `cursor:pointer` на всех кликабельных.
- [ ] Hover-переходы 150–300ms; никаких мгновенных смен состояний.
- [ ] Контраст текста ≥ 4.5:1 (светлая тема).
- [ ] Видимый focus для клавиатуры.
- [ ] `prefers-reduced-motion` уважается.
- [ ] Адаптив: 375 / 768 / 1024 / 1440.
- [ ] Нет горизонтального скролла на мобиле; контент не прячется под фикс-навбаром.

## 8. Админка (`/admin`)

- **Вход:** `/admin/login` — одна учётка; пароль как bcrypt-хэш в `.env`
  (`ADMIN_PASSWORD_HASH`). Сессия — подписанная cookie, тайм-аут.
- **Dashboard:** счётчики (товаров, активных, по категориям), быстрые ссылки.
- **Товары:**
  - Список с поиском/фильтром, тумблер видимости (`is_active`), «избранное»,
    drag-sort или поле порядка.
  - Форма add/edit с **табами языков (el/en/ru/fr)**: title, description, material
    на каждый язык; общие поля (категория, цена, валюта, «цена по запросу», флаги).
  - **Загрузка фото**: мультизагрузка, превью, изменение порядка, выбор главного,
    удаление. Pillow: ресайз до макс. стороны (напр. 1600px), генерация thumbnail
    (напр. 400px), сохранение в WebP, имена — uuid.
- **Категории:** CRUD + переводы названий.
- **Настройки сайта:** контакты, соцсети, координаты карты, язык по умолчанию,
  глобальный тумблер показа цен.
- **Заявки:** в MVP не храним (форма шлёт email/мессенджер). Опция на будущее.

**Безопасность:** CSRF-токены на формах админки, ограничение типов/размера
загружаемых файлов (только изображения), валидация Pillow, экранирование вывода
(Jinja2 автоэкранирование), rate-limit на логин.

## 9. Нефункциональные требования

- **Производительность:** lazy-load изображений, WebP, reserve space (нет CLS),
  лёгкий JS. Целевой LCP < 2.5s локально.
- **Доступность:** alt у фото, aria-labels у иконных кнопок, контраст, фокус, семантика.
- **SEO:** человекочитаемые slug, meta/og на каждый язык, hreflang, sitemap.xml, robots.txt.
- **Адаптивность:** mobile-first, брейкпоинты 375/768/1024/1440.
- **Офлайн-устойчивость (опция):** локальные шрифты и карта-fallback, если нужен показ без интернета.

## 10. Запуск и проверка (verification)

**Запуск (локально):**
```powershell
cd D:\DreamCatcher
python -m venv .venv; .\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env   # задать SECRET_KEY и ADMIN_PASSWORD_HASH
python -m app.seed       # демо-данные (опц.)
python -m uvicorn app.main:app --reload
# http://127.0.0.1:8000  ->  редирект на /el/
```

**Критерии готовности (definition of done):**
- [ ] `/` редиректит на `/el/`; переключатель меняет язык, оставаясь на странице.
- [ ] Каталог и карточка отображают демо-товары с фото и ценой/«по запросу».
- [ ] Все 4 языка работают; отсутствующий перевод даёт фолбэк на el.
- [ ] Админ логинится; добавляет товар с фото и 4 переводами; товар появляется на сайте.
- [ ] Тумблер видимости скрывает/показывает товар.
- [ ] Загрузка фото создаёт превью; неизображения отклоняются.
- [ ] Адаптив проверен на 375/768/1024/1440 (скриншоты).
- [ ] Pre-delivery чек-лист дизайна (§7) пройден.

**Как проверять (по best-practices «дай проверку, которую можно запустить»):**
- Запустить сервер, пройти сценарии выше вручную + скриншоты ключевых страниц.
- Опционально: pytest на роуты (200/редиректы, фолбэк языка, гард админки),
  и `/code-review` свежим субагентом по диффу против этого SPEC.
